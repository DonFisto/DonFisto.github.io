# Robotics Portfolio Prompt Kit

This package contains the documents needed to build and maintain a professional portfolio website for the autonomous-driving perception and local-mapping project.

## Files

- `PORTFOLIO_SPEC.md` — permanent project constitution: verified facts, visual language, content structure, technical constraints, and rules against fabricated claims.
- `RoboticsPortfolioPromptGuide.md` — five large implementation prompts, designed to be run in order.
- `PORTFOLIO_REVIEW_CHECKLIST.md` — review criteria for implementation, PDF export, deployment, and future updates.

## Recommended workflow

1. Add these Markdown files to the root of `DonFisto/DonFisto.github.io`.
2. Run Phase 1 from `RoboticsPortfolioPromptGuide.md`.
3. Review the resulting plan and any updates to the specification.
4. Run Phases 2–5 in order.
5. Use `PORTFOLIO_REVIEW_CHECKLIST.md` before accepting each phase.
6. Keep `PORTFOLIO_SPEC.md` in the repository permanently. Future agents must read it before making changes.

## Core principle

The website must present direct engineering evidence rather than decoration. Real project outputs, limitations, architecture, development decisions, and repository links should carry more weight than visual effects.
